## Commands

This project uses `just` as the command runner (see `Justfile`). Load `.env` variables before running commands.

```bash
just run          # Run app with hot-reload (reflex)
just migrate up   # Apply DB migrations
just migrate down # Rollback DB migrations
just compose up   # Start local dev services (PostgreSQL + pgAdmin)
just compose down # Stop local dev services
just sqlc         # Regenerate sqlc code from SQL queries
just swag         # Regenerate Swagger docs (runs swag init --parseDependency)
go build ./...    # Build
go test ./...     # Test
go vet ./...      # Vet
```

Swagger UI is served at `http://localhost:3000/api/docs/index.html` when the app is running. The spec is at `/api/docs/doc.json`. Uses `swaggo/http-swagger/v2` (a standard `net/http.Handler`) which Fiber v3 adapts natively — no fiber/v2 dependency required.

pgAdmin is available at `http://localhost:5050` when compose is running.

## Architecture

Go 1.25 SaaS licensing/entitlement management platform using Fiber v3, PostgreSQL, and sqlc.

**Layers:**
- `main.go` — wires config, DB, state, and Fiber app
- `internal/config/` — Viper-based config (reads `config.toml` + `CLP_*` env vars)
- `internal/state/` — singleton `AppState` (config + storage, mutex-protected)
- `internal/storage/postgres/` — pgxpool connection with Fibonacci backoff retry
- `internal/database/` — **generated by sqlc**, do not edit manually (`models.go`, `querier.go`, `queries.sql.go`)
- `internal/handlers/` — one file per resource group; import `state.AppState.DB()` for DB access
- `internal/swagger/` — lightweight Fiber v3-native Swagger UI handler (no fiber/v2 dependency)
- `docs/` — **generated by `just swag`**, do not edit manually
- `internal/routes/` — Fiber route registration
- `internal/types/` — shared type definitions
- `sql/migrations/` — schema migrations (golang-migrate)
- `sql/queries/queries.sql` — SQL source for sqlc generation

**Database workflow:** Edit `sql/queries/queries.sql` → run `just sqlc` → commit generated files in `internal/database/`.

**Swagger workflow:** Add/edit swaggo annotations in handler files → run `just swag` → commit generated `docs/` files. The `@BasePath` is `/api`, so `@Router /tenants [get]` maps to `/api/tenants`. Run with `--parseDependency` (already in the Justfile recipe) because models use `pgtype` fields.

**Schema workflow:** Add migration files in `sql/migrations/` (naming: `<timestamp>_<name>.up.sql` / `.down.sql`) → run `just migrate up`.

## Configuration

Reads `config.toml` from current directory or `/etc/clp/`. All keys can be overridden via env vars with `CLP_` prefix and `_` replacing `.` (e.g., `CLP_POSTGRES_HOST`). See `config.toml` for all available keys.

## Key Entities

The domain model centers on multi-tenant licensing: `tenants` → `subscriptions` → `subscription_features` (entitlements). Deployments and `endpoints` (agents) belong to tenants and are tracked for usage metrics. Audit trail via `entitlement_events`.
